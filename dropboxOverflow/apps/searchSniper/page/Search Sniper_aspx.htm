Search Sniper
     

Search Sniper is a SharePoint 2013 App Part. Search Sniper is added to a SharePoint page in the same way a web part is, but the Search Sniper App Part can be acquired from an App Store and runs in an environment that should not impact the host SharePoint Services.

Search Sniper is designed to simplify creating searches against large SharePoint Lists or Libraries. With Search Sniper, you can search against one or more fields within the list and makes use of various operators such as Contains, Greater Than, Equals, etc. You can also combine search criteria using And's and Or's.

Installation Instructions:
  • Go to any web part page in the host web.
  • Edit the page so you can add an App Part.
  • Insert the Search Sniper App Part into your page.
  • Edit the App Part, setting the name of the target list or library in the App Part custom properties.  (SharePoint Configuration group, SharePointList property).  If you do not set a valid libary name you will get an error message "List or Library not found.  Please edit the app part and set the list or library name in the SharePoint Configuration section".

How to Use Search Sniper:
  • If you have entered a valid list or library name in the edit web part configuration step, you should see a group of check boxes corresponding to the searchable fields in your list.  These check boxes allow you to define which fields you'd like to see in your results set.
    • There is a "Check/Uncheck All" check box at the top of the checkbox set.  This allows you to quickly uncheck all the fields in your potential results set and select just the ones you're interested in.   If you select all the fields, you may have to scroll the results set to the right to see all the results.
  • You'll find a search criteria grid under the check boxes.  The grid includes the following fields:
    • Drop down list for Field Names. Should be the same as the names in the pick for display list.
    • Drop down list of supported conditions.  Supported conditions include Contains, Begins With, Equals, Does Not Equal, Greater Than, Greater Than or Equal, Less Than, Less Than or Equal, Is Null, Is Not Null .  Note that there are some restrictions, for example Contains or Begins With cannot be used with Dates, Numbers, Currency, or Calculated fields.
    • Field value for comparison, if needed.
    • Logical And/Or operator, for compound conditions.
    • Delete button, to delete criteria when there are more than one.
    • Add criteria button, for adding additional search criteria.
    • Click to Search, which triggers the search.
  • Use the add criteria button to add more lines the the criteria grid.  You can use the same field on multiple criteria lines, for example you can compare a field greater than or equal to date 1 and less than or equal to date 2.
  • Date comparisons search the date only (day, month, year). Time of day is not searched.
  • The results grid is pagable, showing up to 15 items per page.
  • You can also sort by clicking on column headers in the results grid. Clicking on the same header twice toggles between listing the results in ascending and descending order.
  • Files in document libraries are hyperlinked, so you can access them from the results grid.
